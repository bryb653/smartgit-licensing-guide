## SmartGit 是什么？为什么开发者需要它

SmartGit 是一个可视化 Git 管理工具，核心定位是：**让 Git 的每一个操作都看得见、操得了**，同时不降低专业深度。

它不像 GitHub Desktop 那样为入门者做了很多简化，也不像 Sourcetree 那样在 Mac 上越更新越卡。SmartGit 走的是专业路线：功能全、稳定、跨平台一致。

内置工具包括：Git-Flow 支持、SSH 客户端、文件 Diff 和三路合并工具。不需要额外安装 WinMerge 或其他对比软件，开箱即用。平台集成方面支持 GitHub、GitLab、Bitbucket、Azure DevOps，可以直接在客户端处理 Pull Request 和代码评审评论，不用在浏览器和 IDE 之间来回切换。

一句话总结：SmartGit 是一款把 `git log --all --graph --oneline` 变成可点击分支图的工具，让你把精力放回代码本身。

## SmartGit 有哪些核心功能值得关注

### 分支图与提交历史

SmartGit 的分支可视化图是开发者最常提的亮点之一。提交历史以图形方式展示，分支关系一目了然，支持拖拽进行 rebase 和 cherry-pick 操作。对于经常维护多条功能分支的团队，这个界面能省掉大量脑力。

### 内置冲突解决工具

SmartGit 自带三路合并工具，合并冲突时可以直接在软件内逐段对比和选择，不需要额外安装 Meld 或 KDiff3。编辑器支持自由编辑，而不是简单的"选左还是选右"。

### AI 辅助提交信息（25.1 新功能）

SmartGit 25.1 版本加入了 AI 辅助功能。不是那种到处弹框的做法，而是需要手动开启，每个仓库和 AI 提供商单独配置。实际使用场景主要是：写了半天代码准备提交，让 AI 根据 diff 自动生成 commit message，以及给 stash 自动生成描述，不用再面对一堆没名字的临时存档。

### 跨平台授权统一

买一份授权，Windows、macOS、Linux 全都能用，界面保持一致。对于需要在不同操作系统间切换的开发者，这一点省去了重新适应的成本。

## 如何激活 SmartGit（逐步操作）

购买完成后，系统会自动发送一封包含授权文件附件的邮件。激活流程非常直接：

1. 打开邮件，下载附件中的授权文件（.license 格式）
2. 启动 SmartGit，点击顶部菜单栏的 `Help > Register…`
3. 在弹出窗口中点击右侧按钮，选择刚才下载的授权文件
4. 点击 `Register` 按钮，完成激活

如果邮件没有收到，先检查垃圾邮件文件夹，或者直接登录数码荔枝账号，在"已购软件"页面找到对应订单下载。

激活转移方面：如果换电脑或重装系统，在原设备卸载软件后，授权可以在新设备上重新激活，不需要联系客服。单用户授权最多可以同时激活 5 台设备。

## SmartGit 与其他 Git 工具的对比

这类问题没有标准答案，但可以给几个参考角度：

**vs GitHub Desktop**：GitHub Desktop 免费，上手门槛极低，但功能相对基础，不支持 SVN，和非 GitHub 平台的集成也偏弱。SmartGit 功能完整度更高，适合有更多定制需求的开发者。

**vs Sourcetree**：Sourcetree 免费，但 Mac 版性能问题是老用户的常见抱怨，且不支持 Linux。SmartGit 跨三个平台且保持一致体验，稳定性口碑更好。

**vs GitKraken**：GitKraken 界面更现代漂亮，免费版功能受限，专业版价格按订阅制计费，长期使用成本比一次性买断高。SmartGit 买断制授权从长期来看性价比更高。

**vs 命令行**：两者不矛盾。SmartGit 本身支持调用外部工具，很多人会在复杂操作时切换终端，日常操作用 GUI，各取所长。
